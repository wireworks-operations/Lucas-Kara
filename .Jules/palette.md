## 2026-03-18 - [Accessibility & Mobile Responsiveness]
**Learning:** In a Google-style search interface where the top bar logo and profile icons are absolutely positioned for desktop centering of the search bar, mobile responsiveness requires resetting these elements to `position: static`. This allows Flexbox's `order` and `flex-basis` properties to correctly wrap the search bar to a second line. Additionally, converting non-semantic `<i>`, `<a>`, and `<div>` elements to semantic `<button>` elements with `background: none; border: none;` maintains visual design while enabling full keyboard accessibility.
**Action:** Always check for absolute positioning in headers when implementing mobile wrapping. Prioritize converting interactive icons to semantic buttons with aria-labels even in "static" HTML prototypes.
